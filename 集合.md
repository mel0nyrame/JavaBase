# 集合

- 数组与集合区别
  - 数组固定大小,集合大小可变
  - 数组能存放基本数据,集合只能存放对象

几乎所有集合类都实现了Collection接口,这个接口定义了集合类所拥有的方法

Collection接口里包含了增删改查已经交并补集的操作

## List集合(列表)

- 特点
  - 有序类,从头插到脚,有下标
  - 可以包含重复元素
  - 继承与Collection接口
  - 可直接获取下标,查询某个元素位置

### ArrayList

和数据结构中相同

### LinkedList

和数据结构中相同,不过是双向链表

|列表|特点|
|:--:|:--:|
|ArrayList|更适合随机访问,可以直接获取下标,不适合大量的插入删除|
|LinkedList|不适合随机访问,需要遍历得到元素,适合插入删除|

## Iterator(迭代器)

用于遍历集合中的元素

- 方法
  - boolean hasNext():是否有下一个元素
  - E next():输出下一个元素

>不同集合的实现方式不同

### ListIterator(针对list的迭代器)

- 方法
  - 继承迭代器的方法
  - 从后往前遍历
  - 获取索引
  - 增删改查元素

## Deque和Qeque(队列)

在Java底层均使用LinkedList实现

可以使用Queue<> queue = new LinkedList<>();来实现队列操作

- 方法
  - Queue(单项队列)
    - 在此接口中均有两个方法,一个会报错一个不会报错
  - Dueue(双端队列)
    - 既可以在队尾入列出列也可以在对首入列出列,可当作栈使用(也封装了栈类似的方法)
  - PriorityQueue(优先队列)
    - 可以保证根据次序出列(存放不一定)

## Set集合

- 特点
  - 全部方法均继承Collections接口
  - 不允许出现重复元素(模拟数学中的集合)
  - 不允许随机访问(不允许通过下标访问)
  - 元素存储无序

### HashSet

- 特点
  - 底层使用HashMap实现(元素为key,value为new的Object),顺序不确定
  - 若要维持插入顺序可使用LinkedHashSet(底层使用LinkedHashMap实现)
  - 若需要插入的元素自动排序则可以使用TreeSet

## Map集合(映射)

反映两个元素之间的对应关系,以键值对方式存储

- 特点
  - 不允许出现重复的key,如果出现会把之前的键值对覆盖

### HashMap

- 特点
  - 底层使用哈希表存储,存储顺序不确定
  - 若需要维持插入顺序,可使用LinkedHashMap

#### HashMap底层实现原理

- 原理
  - 底层使用数组存放链表的头节点(Node<K,V>[] table ),位置不确定 (哈希表)
  - 当存放位置相同的时候,相同位置以链表的形式连接
  - 若要维持插入的顺序则使用LinkedHashMap,若要自动排列顺序则使用TreeMap(底层维护了一颗红黑树)

- 特点
  - 初始容量为16,动态扩容,使用负载因子(loadFactor:0.75,代表只要使用的容量大于哈希表长度的75%时就开始扩容,即为12),长度始终为$2^n$
  - 当相同位置的链表过长时,链表会自动转为红黑树,但最终还是要增长哈希表的长度

HashMap的Node源码:

```java
static class Node<K,V> implements Map.Entry<K,V> {
    final int hash; 
    final K key;
    V value;
    Node<K,V> next;

    Node(int hash, K key, V value, Node<K,V> next) {
        this.hash = hash;
        this.key = key;
            this.value = value;
        this.next = next;
    }
}
```

>LinkedHashMap是双向链表,Node节点会多一个保留前节点的指针
